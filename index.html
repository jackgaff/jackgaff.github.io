f-ocean engine

A cross‑platform, 2d C++ engine featuring SDL2 rendering, Box2D physics, Lua scripting, and a robust CMake‑based build pipeline.

Technologies Used

  • C++17
  • CMake 3.16+
  • Lua 5.4
  • LuaBridge
  • SDL2 (+ image / mixer / ttf)
  • Box2D
  • GLM
  • RapidJSON
  • CPack
  • GitHub Actions (optional)

Engine Features

  • Component‑based architecture with runtime Lua scripting
  • Pixel‑perfect 2D physics powered by Box2D
  • Sprite & text rendering via SDL2 with batching for performance
  • Cross‑platform build & packaging system configured entirely via CMake
  • Hot‑reloadable resources for rapid iteration
  • JSON‑driven scene & asset configuration
  • Audio playback for SFX & music using SDL2_mixer

Build & Package with CMake + CPack

The engine ships with an opinionated CMakeLists.txt that hides platform quirks while giving you sensible defaults:

  • Modern warnings & build types (-Wall -Wextra + -g/-O3) baked in.
  • Vendored static libs for Box2D and lua, eliminating external dependencies.
  • Platform‑aware linkage: pkg‑config on Linux, .framework bundles on macOS (with automatic RPATH fix‑up), and .lib/.dll copying on Windows.
  • Post‑build resource copy so your executable always ships with a self‑contained resources/ folder.
  • Install rules that follow OS conventions (share/ on Linux, alongside the binary elsewhere).
  • CPack integration to generate .zip / .tgz ready for end‑users—no extra scripts needed.

Quick Start

# Clone & build
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build . -j$(nproc)

# Run the sample scene
./bin/game_engine

Install Locally

cmake --install . --prefix "$HOME/.local"

Create Distributable Archives

# Inside the build directory
cpack              # Generates game_engine-Linux.tar.gz / .zip etc.

Need an MSI, DEB, or DMG? Swap the generator: cpack -G DEB (after adding install metadata).

Using the Engine in Your Game

  1. Clone the repo and add it as a submodule or drop it alongside your game sources.
  2. Include add_subdirectory(game_engine) in your own CMakeLists.txt.
  3. Link against game_engine: 
    target_link_libraries(MyGame PRIVATE game_engine)

That’s it—CMake propagates include directories, SDL/Box2D linkage, and resource copying for you.
