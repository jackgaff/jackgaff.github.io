C++ 2D Game Engine

A cross‑platform engine featuring SDL2 rendering, Box2D physics, Lua scripting, and a robust CMake‑based build pipeline.

Technologies Used

  • C++17
  • CMake 3.16+
  • Lua 5.4
  • LuaBridge
  • SDL2
    + image / mixer / ttf
  • Box2D
  • GLM
  • RapidJSON
  • CPack
  • GitHub Actions
    (optional)

Engine Features

  • Component‑based architecture with runtime Lua scripting
  • Pixel‑perfect 2D physics powered by Box2D
  • Sprite & text rendering via SDL2 with batching for performance
  • Cross‑platform build & packaging system configured entirely via CMake
  • Hot‑reloadable resources for rapid iteration
  • JSON‑driven scene & asset configuration
  • Audio playback for SFX & music using SDL2_mixer

Building from Source

Windows

git clone https://github.com/jackgaff/game_engine.git
cd game_engine
mkdir build && cd build
cmake -G "Visual Studio 17 2022" ..
cmake --build . --config Release
REM Install & package (no sudo on Windows)
cmake --build . --target install --config Release
cpack

macOS

git clone https://github.com/jackgaff/game_engine.git
cd game_engine
mkdir build && cd build
cmake ..
cmake --build . --config Release
# Optional: generate an Xcode project
# cmake -G Xcode ..
# Install & package
sudo cmake --build . --target install --config Release
sudo cpack

Linux

git clone https://github.com/jackgaff/game_engine.git
cd game_engine
mkdir build && cd build
cmake ..
make -j$(nproc)
# Install system‑wide (uses DESTDIR under sudo)
sudo make install
# Or with CMake
sudo cmake --build . --target install
# Package into .tar.gz / .zip
sudo cpack

What does cpack do?

cpack leverages the install rules in CMakeLists.txt to bundle the engine—binaries, resources, and licenses—into platform‑appropriate archives (default: .tgz & .zip). Run it in your build directory after install, and you’ll get ready‑to‑ship packages in _CPack_Packages/.
